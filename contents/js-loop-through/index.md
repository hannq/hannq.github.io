# 深究一下js中的循环、遍历！

###### 碎碎念

> 坐标南京, 年底**项目不是很多**(~~工作不饱和~~), **心血来潮**(~~脑子抽了~~)**去面了一波试**(~~看看自己是否还有竞争力~~)，但是面试的体验不是特别好, 具体哪家公司就不点名了, 毕竟大家都是要恰饭的嘛。
>> 面试官: 说说 `css 盒子模型` 呢？<br>
>> **我**: `css 盒子模型` 分两种, `ie盒模型`、`标准盒模型` 巴拉巴拉 ...<br>
>> 面试官: 你说的不对, 应该是 `padding` 是内边距、`margin` 是外边距 ...<br>
>> **我**: ???<br>
>> 面试官: `for in` 比较熟吗? 说说 `for in` 的东西是什么呢？<br>
>> **我**: 比较熟吧, `in` 关键字是 `js` 中一个非常古老的关键字, 可以用来遍历所有可枚举的属性名, 巴拉巴拉 ...<br>
>> 面试官: 你又说错了, `for in` 可以遍历的是 对象！<br>
>> **我**: ???<br>
>> 面试官: 那你说说 `for of` 遍历的是对象的 键还是值呢?<br>
>> **我**: `for of` 是 es6 新加的语法, 可以用来遍历可迭代对象, 所有实现了 `[Symbol.Symbol.iterator]` 方法的对象都可以通过 `for of` 遍历出对应的值, 所以对象的 键/值 能否被遍历出来取决于 `[Symbol.Symbol.iterator]` 方法如何定义, 巴拉巴拉 ...<br>
>> 面试官: 你还是说错了, `for of` 遍历的是值，`for in` 遍历的是 键, 这就是两者的区别, 不信你回去可以用数组试一试。<br>
>> **我**: (站起身)不好意思，我能力有限, 可能不适合这个岗位。<br>

> 回来的路上, 我就在想, 当时我应该怎么解释 `for of` 和 `for in`, 才可以面试官理解呢? 或者说如何我要怎么系统地解释常见的对象在 `for of` 下的表现的缘由, 让面试官理解, 并成功挽救下一个被面试者的面试体验呢。要知道一个好的面试体验可以让被面试者在面试中受益良多, 而差的面试体验甚至让人觉得公司不尊重被面试者, 让被面试者感觉就像吃了 `奥利给` 一样, 浑身难受。于是我成功的被自己提出的问题难住了, 我并不知道常见对象的迭代器接口实现的具体细节, 痛定思痛, 我决定把这些知识点都搞懂, 于是就有了下面这篇文章。
